# AS 渲染自检

用户要审查一份 AS 渲染输出时：

1. 先确认拿到三份输入：**原文、模板、待测输出**。缺哪份就按协议里的降级规则标注跳过，不要自己脑补缺失的材料。
2. 读取 `references/as-check-protocol.md`，**严格按其中的清单、评分、报告格式、报告纪律执行**。不要自行增减检查项或改动评分权重。
3. 产出 MD 报告，与输出文件同目录，文件名 `<输出文件名>.check-<评测模型>-<时间戳>.md`。
   - **时间戳取自系统时间**（运行 `date +%Y%m%d%H%M`），不要凭空估算。
   - 取不到系统时间就省略时间戳；若同名文件已存在，末尾加 `-02`、`-03`。

`references/as-check-protocol.md` 同时是可独立分发的协议文件：可原样上传给其他平台 / 模型做交叉评测，无需本 skill。多模型部署与结果聚合的方法见 `USAGE.md`（那是给使用者的，本 skill 执行时不必读）。
