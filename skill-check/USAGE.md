# AS 自检 —— 运行说明（给使用者，模型不读）

## 三种用法

### 1. 作为 skill（Claude Code / 支持 skill 的环境）

装好后直接说「帮我检查 / 评测这份 AS 渲染输出对不对」即可自动触发。
准备好三份输入：原文、模板、待测输出。报告会写到输出文件同目录下。

### 2. 上传到 claude.ai / chatgpt.com 等在线平台

上传 `as-check-protocol.md` + 你的**原文** + **模板** + **待测输出**，然后粘贴下面这段提示词：

```
我上传了几份文件：一份《AS 渲染自检协议》、一份原文、一份 AS 模板、一份待检测的渲染输出。

请严格按协议执行检测，不要自由发挥：
- 对照原文检查内容保真（A 组），对照模板检查规则遵守（B 组），逐条核对协议清单里的每一项。
- 缺哪份材料，就按协议的降级规则标注跳过，不要脑补。
- 按协议的评分规则打分，完全按协议的报告格式产出：先报头、评分细则、问题清单、问卷调查。
- 报告纪律：只报有把握的问题，每条带位置和一处证据，原因 / 修复 / 防复发各一行，不展开议论；通篇没问题就写「未发现 P0/P1」。
```

（平台不便传文件时，把四份内容依次贴在提示词下方、标清哪段是哪份即可。）

### 3. 交叉评测（推荐做法）

同一份输出，用 **≥2 个模型**各跑一遍；尽量选**不同家族、且不同于生成它的那个模型**（避免模型替自己辩护）。

聚合各份报告，按「编号」归并：

- 多数命中 = **确认项**，直接修。
- 出现分歧 = **争议项**，人来看（归因不一致也算，往往说明问题两头都占）。
- 仅一个模型报 = **孤立项**，人来看（多半误报，偶尔是神来之笔）。

**扣分只算确认项。**

## 可比性前提

只有在「同一输入 + 同一清单 + 同一权重 + 只扣确认项」时，平台 / 模型之间的总分才真能比。
换了输入，就只比违规率（每 N 个组件几个违规），不比绝对分。

## 怎样把报告变成 AS 改进（核心）

目标是用报告反过来**升级模板，把问题挡在发生之前**。

- **模板改进清单 ＝ 所有非「无」的「防复发」条目 ＋ 问卷 Q3 的答案**，跨轮单独攒。这是你改模板与头部说明的直接依据，比单次问题报告更有长期价值。
- 按**归因**看分布、定策略：`模板表述` 多 → 重写头部说明；`模型固有` 多 → 除了加 spec 护栏降频，还得考虑后处理脚本或换模型（spec 兜不住）；`内容歧义` 多 → 给映射加指引。
- 报头里模板文件名带版本号；改完头部说明再跑一遍，靠版本号做**回归追踪**——判断上次改动到底修没修掉问题。
